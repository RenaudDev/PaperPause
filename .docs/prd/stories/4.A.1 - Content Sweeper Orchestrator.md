# 4.A.1 — Content Sweeper Orchestrator ✅

Epic: `4 - Collection Content Engine`

## Goal
Implement the core logic to scan collections, plan content, and assemble 2,000-word articles using sectional chaining.

## Scope
-   Create `scripts/morning-routine/tasks/write-collection-content.ts`.
-   Implement scanning logic for `_index.md` files targeting word count < 2000.
-   Implement `getStyleMap(dir)` for asset discovery and style grouping.
-   Implement the **Deterministic Outline** sectional generation loop.

## Acceptance criteria
-   Scanner correctly identifies folders needing content enrichment.
-   Visual audit detects artistic styles and selects representative images.
-   Sectional generation passes summary context from section N to N+1 correctly.
-   Assembly produces a valid Markdown buffer containing all 7 required sections.
-   Writes back to `_index.md` with `content_generated: true` and `last_swept` metadata.

## Rollback
-   Disable `ENABLE_CONTENT_SWEEP` flag.
-   Revert `_index.md` files using Git if necessary.
