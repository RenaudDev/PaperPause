# 3.0.2 — The Expandable Vault

**Epic**: `3 - Collection UX Overhaul`

## Goal
Implement soft pagination that caps initial view to 24 items while ensuring the full vault is searchable via filters and expandable on demand.

## Scope
- **Template Changes (`list.html`)**:
  - Iterate over collection items; for items 1-24, render normally.
  - For items 25+, add `class="hidden"` and `loading="lazy"` to the `<img>` tag.
  - Insert "Show All" button after item #24 (only if collection has >24 items).

- **JavaScript (`filter.js` or new `vault.js`)**:
  - **"Show All" Button Handler**:
    - Remove `hidden` class from all vault items.
    - Hide the "Show All" button itself.
  - **Filter Force-Expand**:
    - When any filter is selected, call `forceExpandVault()` before filtering.
    - `forceExpandVault()` removes `hidden` class from all items and hides "Show All" button.
  - Export `forceExpandVault()` for use by filter module.

- **CSS Updates**:
  - Add `.hidden { display: none; }` utility class.
  - Style "Show All" button to match site design system.
  - Add reveal transition (optional): fade-in for newly visible items.

## Acceptance criteria
- [ ] Collection pages with ≤24 items render normally without "Show All" button.
- [ ] Collection pages with >24 items show exactly 24 items initially; remaining are hidden.
- [ ] Hidden images have `loading="lazy"` attribute.
- [ ] "Show All" button is visible and positioned correctly after last visible item.
- [ ] Clicking "Show All" reveals all hidden items and hides the button.
- [ ] Selecting any filter automatically expands vault before applying filter logic.
- [ ] Core Web Vitals are not degraded (LCP, CLS, INP remain green).

## Rollback
- Revert template changes in `list.html`.
- Revert JavaScript changes in filter/vault modules.
- Revert CSS additions for `.hidden` class and "Show All" button.
