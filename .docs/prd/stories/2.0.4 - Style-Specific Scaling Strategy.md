# 2.0.4 - Style-Specific Scaling Strategy

**Epic**: `2 - Content Quality & Scaling`

## Goal
Prove the "Style-Specific Collection" pattern to enable unlimited niche scaling by separating generic subjects into style-locked collections.

## Scope
- **Create "Steampunk Dogs"**:
    - Clone `animals-dogs.json` → `animals-dogs-steampunk.json`.
    - Lock `attributes.styles` to `["Steampunk"]`.
    - Update `base` prompt to explicitly mention "Steampunk Vector line art".
- **Create "Kawaii Dogs"**:
    - Clone `animals-dogs.json` → `animals-dogs-kawaii.json`.
    - Lock `attributes.styles` to `["Kawaii"]`.
- **Validate**: Ensure the system generates two distinct folders/RSS feeds (`dogs-steampunk` and `dogs-kawaii`) during the next run.

## Acceptance criteria
- [ ] `animals-dogs-steampunk.json` and `animals-dogs-kawaii.json` exist in `config/prompts/`.
- [ ] Daily generation run produces images in `dogs-steampunk` and `dogs-kawaii` folders.
- [ ] Generated images adhere to their respective style locks (no style whiplash).

## Rollback
- Delete `animals-dogs-steampunk.json` and `animals-dogs-kawaii.json`.
