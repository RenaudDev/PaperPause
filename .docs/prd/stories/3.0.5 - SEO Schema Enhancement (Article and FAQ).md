**Epic**: `3 - Collection UX Overhaul`

## Goal
Inject `Article` and `FAQPage` schema into the collection page's JSON-LD graph to maximize visibility for Authority content and Rich Results (FAQ snippets).

## Scope

### Partial Changes (`seo/schema.html`)
- **Article Node**:
    - Trigger: `if .Truncated` (content exists after `<!--more-->`).
    - Fields: `@type: Article`, `headline: .Params.h1`, `articleBody: .Content | plainify`, `author: @id: organization`.
- **FAQPage Node**:
    - Trigger: Detection of FAQs in the content or `.Params.faqs`.
    - Fields: `@type: FAQPage`, `mainEntity: [ Question nodes ]`.
    - Content Parsing: Must extract text from the markdown or use structured frontmatter to populate the schema.

## Acceptance criteria
- [ ] Collection pages with long-form content contain an `Article` schema node in the JSON-LD graph.
- [ ] Collection pages with FAQs contain an `FAQPage` schema node.
- [ ] Schema passes validation via Schema Markup Validator / Rich Results Test.
- [ ] No schema is injected if the relevant content sections are missing (no bloat).
