# 3.0.4 — Visual Index

**Epic**: `3 - Collection UX Overhaul`

## Goal
Create a dense, visually appealing grid of square tiles for parent collection pages (e.g., `/animals/`) that maximizes above-the-fold density and collection discovery.

## Scope

### Template Changes (`list.html`)
The existing template already handles subsections differently (lines 33-39):
```html
{{ if $hasSubsections }}
<div class="grid grid-cols-2 lg:grid-cols-4 gap-4 lg:gap-6">
    {{ range .Sections }}
    {{ partial "collection-card.html" . }}
    {{ end }}
</div>
```

Modify `collection-card.html` OR create new `visual-index-tile.html` partial:
- Render child collections as **1:1 aspect ratio square tiles**.
- Each tile includes:
  - Background image (from collection's `_index.md` `image` frontmatter or representative child image).
  - Semi-transparent overlay for text legibility.
  - Collection title as text overlay.
  - Hover effect (slight zoom, overlay darkening).
- Link each tile to its collection page.

### CSS Updates
- Square tile aspect ratio: `aspect-square` (1:1).
- Background image cover: `bg-cover bg-center`.
- Text overlay: semi-transparent dark background, white/light text.
- Responsive grid:
  - Desktop: 4 columns (`lg:grid-cols-4`).
  - Tablet: 3 columns (`md:grid-cols-3`).
  - Mobile: 2 columns (`grid-cols-2`).
- Hover transitions: scale, overlay opacity.

### Image Source Logic
For each collection tile, determine background image:
1. If collection `_index.md` has `image` frontmatter → use that.
2. Else if collection has child pages → use first child's `image_url`.
3. Else → use placeholder or gradient background.

## Acceptance criteria
- [ ] Parent collection pages (e.g., `/animals/`) render child collections as square tiles.
- [ ] Each tile displays collection title as text overlay.
- [ ] Each tile has a background image from the collection.
- [ ] Tiles maintain 1:1 aspect ratio across all breakpoints.
- [ ] Clicking a tile navigates to the target collection.
- [ ] Hover effect provides visual feedback (zoom, overlay change).
- [ ] Leaf collection pages continue to render the standard gallery (not Visual Index).
- [ ] Layout is responsive and visually balanced.

## Rollback
- Revert `collection-card.html` partial changes OR remove `visual-index-tile.html`.
- Revert associated CSS changes.
