## Goal
Integrate the Scheduler and Conductor into the GitHub Actions lifecycle to ensure 24/7 autonomous distribution.

## Scope
- Implement `.github/workflows/distribution-system.yml`.
- **Jobs:**
  - `scheduler`: Runs at `00:00 UTC` to build the queue.
  - `distributor`: Runs hourly at minute 45 to distribute one item.
- Ensure proper permissions (`contents: write`) for committing queue updates.
- Use `workflow_dispatch` for manual testing.

## Acceptance criteria
- Workflow triggers correctly on a schedule.
- `scheduler` job commits the new `distribution-queue.json`.
- `distributor` job executes successfully and uses the repo secret `MAKE_WEBHOOK_URL`.
- Hourly distribution avoids collisions and double-posting.
